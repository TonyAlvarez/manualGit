PhpStorm, IntelliJ y derivados

Aquí veremos como usar las funciones básicas de Git desde PhpStorm, IntelliJ o derivados. Haz scroll hacía abajo o elige la instrucción en la lista para saltar directamente a ella.

Git init

Crea un nuevo repositorio en el directorio actual.

¿Cómo se hace?

Una vez que tenemos creado el proyecto en PhpStorm, IntelliJ o derivados.

GitHub

En la barra de herramientas seleccionamos VCS/Import into Version Control/Share project on GitHub.

Git

En la barra de herramientas seleccionamos VCS/Import into Version Control/Create Git Repository..., nos aparecerá una ventana en la que debemos seleccionar el proyecto en el cual queremos crear el repositorio.

Git clone

Clona un repositorio existente en el directorio actual.

¿Cómo se hace?

Una vez que abrimos el PhpStorm sin ningún proyecto abierto seleccionamos la opción Check out from Version control

Nos aparece una ventana en la que debemos seleccionar el repository GitHub que tenemos compartidos o creados en nuestra cuenta. Una vez seleccionado indicamos la ruta donde queremos crear el proyecto y seleccionamos Clone.

Git add

Pone uno o varios archivos en seguimiento.

¿Cómo se hace?

Para añadir un archivo, en el explorador del proyecto seleccionamos el archivo con le botón secundario del ratón y seleccionamos Git/Add.

Después

Git reset

Elimina el seguimiento a uno o varios archivos, hace exactamente lo contrario que Git add.

¿Cómo se hace?

Para el reset, en la barra de herramientas VCS/Git/Reseat HEAD....

Git revert

Genera automáticamente un nuevo commit que deshace los cambios de un commit anterior, de esta forma se evita perder esos cambios del historial. Se pierden los cambios realizados después del último commit.

¿Cómo se hace?

  • Una manera de hacer un revert sería en la parte derecha de la barra superior pulsando el icono de una flecha de color violeta.
  • Otra manera sería en la barra de herramientas VCS/Git/Revert.

Git commit

Introduce las instantáneas de los archivos del área de preparación (staging) en el historial del control de versiones.

¿Cómo se hace?

  • Una manera sería seleccionar con el botón secundario del ratón el archivo en el explorador del proyecto y seleccionamos la opción Git/Commit File...
  • Otra manera sería seleccionar en la derecha de la barra superior seleccionar el icono con la flecha verde, el primer procedimiento será crear el commit y posteriormente nos permitirá realizar un push.

Git log

Lista el historial de commits del repositorio mostrando el autor, la fecha y su descripción.

¿Cómo se hace?

Para ver el log del proyecto en la barra de herramientas seleccionamos VCS/Git/Show History.

Git branch

Lista todas las ramas locales que hay en el repositorio.

¿Cómo se hace?

En la parte inferior a la izquierda aparece la opción Git:[ramaActual], si no tenemos creada ninguna aparecerá Git: Master. Clicamos encima de eso y seleccionamos la opción New Branch.

Aparecerá una ventana en la que debemos introducir el nombre que deseamos que tenga la nueva rama, y una vez creada ya nos desplaza a esa rama.

Git checkout

Actualiza el directorio de trabajo, actualizando los archivos locales con el archivo, el commit o la rama especificada.

¿Cómo se hace?

Para cambiar de rama, en la parte inferior a la izquierda, en el mismo lugar donde creamos las ramas, seleccionamos la opción Git:[ramaActual]. Aparecerán todas las ramas creadas y tan solo debemos seleccionar a la que deseamos desplazarnos.

Git fetch

Descarga los últimos cambios desde el repositorio remoto, no incorpora lo cambios en los archivos locales.

¿Cómo se hace?

Para realizar un fetch seleccionamos en la barra de herramientas la opción VCS/Git/fetch.

Git push

Envía los commits pendientes al repositorio remoto.

¿Cómo se hace?

Hay diferentes maneras de hacer un push:
  1. Si deseamos hacer un commit y un push a la vez seleccionamos el icono con una fecha de color verde que aparece en la barra superior.
  2. Si tan solo deseamos realizar un Push para ello debemos desplazarnos a la barra de herramientas VCS/Git/Push.

Git pull

Descarga la última versión del proyecto desde el repositorio remoto e incorpora los cambios al repositorio local.

¿Cómo se hace?

Seleccionamos el icono que contiene una flecha azul de la barra superior o bien vamos a la barra herramientas VCS/Git/Pull

Git merge

Cambina los cambios de la rama especificada o de un repositorio remoto en el directorio de trabajo o en otra rama.

Git merge es una acción no destructiva, por lo que es más recomendable para usuarios noveles. Como inconveniente, al usar merge en el historial de commits veremos los commits de las dsintintas ramas por separado, y el commit en el que hacemos merge saldrá como un commit especial que une las dos lineas.

¿Cómo se hace?

Para realizar un merge, en la barra de herramientas en VCS/Git/Merge Changes... O bien a la hora de hacer un pull nos pregunta si deseamos hacer un Merge.

Git rebase

Cambina los cambios de la rama especificada o de un repositorio remoto en el directorio de trabajo o en otra rama.

Git rebase, como su nombre indica, rebasa la rama y todo su historial en la rama que le especifiquemos. El primer commit de la rama se mostrará después del último commit de la rama en la que rebasemos, de esta forma la historial gráfico de commits es mucho más sencillo de leer:

El inconveniente de rebase es que puede ser un comando destructivo si lo usamos de forma inadecuada. La regla que siempre hay que cumplir es no rebasar nunca la rama principal del proyecto en una rama secundaria.

¿Cómo se hace?

Para realizar un rebase, en la barra de herramientas en VCS/Git/Rebase.
