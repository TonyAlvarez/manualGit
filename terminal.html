Terminal

Aquí veremos como usar las funciones básicas de Git desde la terminal.

Haz scroll hacía abajo o elige la instrucción en el menú flotante para saltar directamente a ella.

Git init

Crea un nuevo repositorio en el directorio actual.

Parámetros

git init [nombreRepositorio] - Le podemos pasar cómo parámetro el nombre del repositorio

Git clone

Clona un repositorio existente en el directorio actual.

Parámetros

git clone [urlRepositorio] - Le pasamos cómo parámetro la URL del repositorio que queremos clonar

Git status

Muestra el estado del proyecto local respecto al último commit (HEAD).

Si hay algún archivo nuevo o modificado lo lista.

Git add

Pone uno o varios archivos en seguimiento.

Parámetros

git add [archivo] - Pasamos el archivo que deseamos seguir.

Git reset

Elimina el seguimiento a uno o varios archivos, hace exactamente lo contrario que Git add.

Parámetros

git reset [archivo] - Le pasamos el archivo al que le deseamos quitar el seguimiento.

Parámetro [mode]

Con el parámetro [mode] cambia por completo su proposito, el modo por defecto es --mixed

git reset --hard [commit] - El modo --hard descarta todos los cambios de todos los archivos y vuelve al commit que le indiquemos.

Git revert

Genera automáticamente un nuevo commit que deshace los cambios de un commit anterior, de esta forma se evita perder esos cambios del historial.

Parámetros

git revert [commit] - Pasamos como parámetro el commit del cual queremos deshacer los cambios

Git commit

Introduce las instantáneas de los archivos del área de preparación (staging) en el historial del control de versiones.

Parámetros

git commit -m "[descripcion]" - Introducimos un mensaje que describa los cambios realizados en está última actualización

Git log

Lista el historial de commits del repositorio mostrando el autor, la fecha y su descripción.

Parámetros

git log --follow [archivo] - Lista el historial de commits que afectan a un archivo en concreto, incluso aunque el archivo haya sido renombrado.

Git branch

Lista todas las ramas locales que hay en el repositorio.

Parámetros

git branch -m [nombreRama] - Crea una nueva rama

git branch -d [nombreRama] - Elimina una rama

Git checkout

Actualiza el directorio de trabajo, actualizando los archivos locales con el archivo, el commit o la rama especificada.

Parámetros

git checkout [commit] - Actualiza todos los archivos del directorio de trabajo al estado del commit especificado.

git checkout [commit] [archivo] - Actualiza el archivo especificado a la instantanea del commit especificado. Los cambios que no se hayan enviado serán perdidos.

git checkout [nombreRama] - Cambia el directorio de trabajo a la rama especificada.

Git fetch

Descarga los últimos cambios desde el repositorio remoto, no incorpora lo cambios en los archivos locales.

Parámetros

git fetch [nombreRemoto] - Se puede especificar el repositorio remoto desde donde se descargarán los cambios, si no se especifica se descargan desde el repositorio por defecto que se haya configurado previamente.

Git push

Envía los commits pendientes al repositorio remoto.

Parámetros

git push [nombreRama] - Envia los cambios de la rama especificada.

Git pull

Descarga la última versión del proyecto desde el repositorio remoto e incorpora los cambios al repositorio local.

Parámetros

git pull [nombreRemoto] - Se puede especificar el repositorio remoto desde donde se descargarán los cambios, si no se especifica se descargan desde el repositorio por defecto que se haya configurado previamente.

git pull [--rebase] - Se cambia el modo de combinar los cambios en los archivos locales a rebase, por defecto es merge.

Git merge

Cambina los cambios de la rama especificada o de un repositorio remoto en el directorio de trabajo o en una otra rama.

Parámetros

git merge [ramaPrincipal] - Le pasamos la rama con la que queremos combinar la rama actual

git merge [ramaPrincipal] [ramaACombinar] - Le pasamos ambas ramas como parámetros, primero la rama principal y segundo la rama que queremos combinar en ella.

Git merge es una acción no destructiva, por lo que es más recomendable para usuarios noveles. Git rebase

Cambina los cambios de la rama especificada o de un repositorio remoto en el directorio de trabajo o en una otra rama.

Parámetros

git rebase [nombreRama] - La rama en la que queremos rebasar los cambios

Git rebase, como su nombre indica, rebasa la rama y todo su historial en la rama que le especifiquemos. El primer commit de la rama se mostrará después del último commit de la rama en la que rebasemos, de esta forma la historial gráfico de commits es mucho más sencillo de leer:

El inconveniente de rebase es que puede ser un comando destructivo si lo usamos de forma inadecuada. La regla que siempre hay que cumplir es no rebasar nunca la rama principal del proyecto en una rama secundaria.
